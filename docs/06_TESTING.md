**التاريخ:** 2026-06-30

## حالات الاختبار الأساسية

### T01: الحساب الأساسي
```
الإجراء: أدخل 100 في فئة 200 جنيه
المتوقع: الإجمالي = 20,000 EGP
          التلميح = "1 رزمة"
```

### T02: رزمة ناقصة
```
الإجراء: أدخل 70 في فئة 200 جنيه
المتوقع: الإجمالي = 14,000 EGP
          التلميح = "70 ورقة (رزمة ناقصة)"
```

### T03: رزم متعددة في فئة واحدة
```
الإجراء: أدخل 250 في فئة 200
المتوقع: الإجمالي = 50,000 EGP
          التلميح = "2 رزمة + 50 ورقة"
```

### T04: إضافة رزمة (زر +)
```
الإجراء: اضغط + في فئة 200، أدخل 70 في الصف الجديد
المتوقع: صف جديد يظهر
          مجموع المجموعة = (250 × 200) + (70 × 200) = 64,000
```

### T05: نص المشاركة الصحيح
```
البيانات:
  فئة 200: صف1=250, صف2=70
  فئة 100: صف1=320

المتوقع في نص المشاركة:
  2x200x100
  1x200x50
  1x200x70
  3x100x100
  1x100x20
```

### T06: التبديل بين البروفايلات
```
الإجراء:
  1. في EGP: أدخل 100 في فئة 200
  2. بدّل لـ USD
  3. أدخل 50 في فئة 100
  4. ارجع لـ EGP

المتوقع:
  - في EGP: الإجمالي = 20,000 EGP (كما تركته)
  - في USD: الإجمالي = 5,000 USD
  - لا خلط بين البروفايلات
```

### T07: الحفظ بعد إغلاق التطبيق
```
الإجراء:
  1. أدخل أرقاماً
  2. أغلق التطبيق تماماً (من الـ recent apps)
  3. أعد فتحه

المتوقع: نفس الأرقام موجودة
```

### T08: إعادة الضبط
```
الإجراء:
  1. أضف فئة طارئة 5 جنيه، أدخل 30
  2. أدخل 100 في فئة 200
  3. اضغط "إعادة ضبط" وأكد

المتوقع:
  - فئة 200: حقل فارغ (0)
  - فئة 5 جنيه الطارئة: لا تزال موجودة لكن حقلها فارغ
  - الإجمالي = 0
```

### T09: الفئة الطارئة
```
الإجراء:
  1. اضغط "+ إضافة فئة"
  2. أدخل القيمة: 5، الاسم: "5 جنيه"
  3. أدخل 40 في الحقل

المتوقع:
  - تظهر في آخر القائمة بعلامة ⚡
  - الإجمالي يشمل 5 × 40 = 200
  - بعد إغلاق التطبيق تبقى موجودة
```

### T10: السجل والمشاركة
```
الإجراء:
  1. أدخل أرقاماً
  2. اضغط زر المشاركة (📤)
  3. شارك عبر واتساب أو نسخ

المتوقع:
  - يُفتح Share Sheet
  - النص بالتنسيق الصحيح
  - يُحفظ تلقائياً في السجل
  - يظهر في HistoryScreen
```

### T11: حالة حافة — صفر
```
الإجراء: اترك جميع الحقول 0 أو فارغة
المتوقع:
  - الإجمالي = 0
  - نص المشاركة يحتوي فقط رأس وذيل (لا سطور فئات)
```

### T12: أرقام كبيرة
```
الإجراء: أدخل 9999 في فئة 200
المتوقع:
  - الإجمالي = 1,999,800 EGP
  - يُعرض بشكل مناسب (لا يتجاوز العرض)
  - في المشاركة: 99x200x100 + 1x200x99
```

## اختبارات الواجهة

### T-UI01: RTL
- جميع النصوص تتجه لليمين
- الأيقونات في الأماكن الصحيحة
- لا نص ينقطع أو يختفي

### T-UI02: الوضع الداكن
- قابل للقراءة في الوضع الداكن
- لا ألوان بيضاء على خلفية بيضاء

### T-UI03: أحجام الشاشات
- يعمل على شاشة 5 بوصة (صغيرة)
- يعمل على شاشة 6.7 بوصة (كبيرة)
- لا overflow في أي مكون

### T13: مقاومة تلف البيانات
```
الإجراء (محاكاة):
  1. أغلق التطبيق
  2. عدّل قيمة 'profiles' في SharedPreferences لتكون نصاً غير صالح
     (مثلاً: "{broken json")
  3. أعد فتح التطبيق

المتوقع:
  - التطبيق يفتح بدون تعطّل
  - يعود للبروفايلات الافتراضية (EGP, USD)
  - لا رسالة خطأ مرئية للمستخدم
  - السجل والجلسات تبدأ فارغة إن كانت هي أيضاً تالفة

ملاحظة: يتطلب محاكاة هذا الاختبار جهازاً rooted أو استخدام Android Studio DevTools
```
